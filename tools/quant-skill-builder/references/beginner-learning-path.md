# 零基础量化学习路径

> 配合 Quant Skill Builder 使用。构建器会根据你的经验自动进入对应路径。

## 路径 A：完全零基础

见下方「第 0-4 周」计划。构建器只提供 3 个固定策略：双均线、RSI 超卖、定投。

## 路径 B：有投资经验，无交易体系

1. 在构建器选择「高阶策略模板」
2. 回测理解策略逻辑，再微调参数
3. 建立 written rules（书面规则）后再模拟盘
4. 参考 [strategy-templates.md](strategy-templates.md) 高阶章节

## 路径 C：成熟交易体系

1. 用自然语言完整描述现有体系
2. AI 输出结构化 spec，你确认后再编码
3. 专业全检：CPCV + DSR + PBO + OOS 衰减比
4. 模拟盘验证与回测偏差 < 30%

---

## 第 0 周：建立正确预期（路径 A）

- [ ] 理解：大多数回测好看的策略，实盘会打折
- [ ] 理解：量化 ≠ 稳赚，是「用规则代替情绪」
- [ ] 阅读 [glossary.md](glossary.md) 全部术语
- [ ] 在构建器中选择「零基础模式」完成第一次配置

## 第 1 周：第一个回测

**目标**：跑通「数据 → 信号 → 回测 → 看结果」，不追求盈利。

1. 选模板：**指数双动量趋势** 或 **指数 Regime 波段**
2. 选市场：A股（510300/沪深300）或 美股（SPY）
3. 风险偏好：2 分（稳健）
4. 导出 Skill，让 AI 搭建项目
5. 跑 **至少 2 年**回测，**必须对比指数买入持有**（Alpha、MaxDD）
6. 看懂输出：总收益、最大回撤、相对基准超额

**通过标准**：能解释回测报告里每个数字的含义。

## 第 2 周：理解过拟合

**目标**：亲眼看到「调参把策略调废了」。

1. 同一个策略，试 10 组不同参数
2. 对比哪组在「没参与调参的数据」上表现最好
3. 运行 DSR/PBO 检验（Skill 中 validation/ 目录）
4. 阅读 [overfitting-validation.md](overfitting-validation.md) 白话版

**通过标准**：能用自己的话解释「为什么不能只信回测最高收益」。

## 第 3 周：换一个策略类型

从以下选一个（与第 1 周不同）：

- 指数 Regime 波段
- 指数智能定投
- 布林带均值回归（advanced，须 Regime 过滤）

对比两种策略在不同市场环境（牛/熊/震荡）的表现。

## 第 4 周：模拟盘

1. 选 audit PASS 的策略
2. 接模拟盘接口（VnPy 模拟 / Freqtrade dry-run）
3. 观察至少 2 周：滑点、信号延迟、实际成交

**通过标准**：模拟盘收益与回测预期偏差 < 30%，或能解释偏差原因。

## 禁止事项（零基础）

- ❌ 跳过回测直接实盘
- ❌ 使用马丁格尔策略实盘
- ❌ 忽视交易成本（佣金、滑点）
- ❌ 在单一参数上过度优化
- ❌ 用少于 2 年数据得出结论

## 推荐首学项目组合

| 组件 | 零基础首选 | 原因 |
|------|------------|------|
| 语言 | Python | 生态最大，AI 辅助最好 |
| 框架 | Backtrader | 概念清晰，文档多 |
| 数据 | AKShare | 免费、安装简单 |
| 策略 | 指数双动量 / Regime 波段 | 逻辑直观 + 对标指数 |
| 验证 | oos-lab | 轻量，DSR/PBO 够用 |
